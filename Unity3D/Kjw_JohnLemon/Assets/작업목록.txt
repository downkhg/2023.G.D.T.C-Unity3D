[게임컨텐츠]
연출의 중요도를 이야기하기위해서 기능도 비교적 간단한 호러장르를 선택함.

개발시에는 3인칭시점을 활용하여 전반적인 게임의 상황을 인지하는 것이 중요하다.
호러 장르이지만 3인칭으로 개발하고 추후 1인칭으로 전환한다.

출발지에서 목적지로가는데 장애물이 없으면 게임이 심심하다.
가고일 -> 이동을 하지못하는 적 -> 히트박스에 대한 이해.
유령 -> 이동하는 적, 특정위치를 순찰한다. -> 이동시키는 방법. 네비게이션사용방법 익히기.

케릭터는 wasd와 방향키를 이용하여 이동한다.(인풋엑시스)
애니메니메이터를 이용하여 애니메이션을 설정한다.(애니메이터)

[기믹추가예정]
총알구현 > (총알)? (총)에서 발사되는 것 -> 총: (총알)을 (앞)으로 밀어내는 것
->총알이 앞으로 이동한다. 
->(총알:게임오브젝트)에 힘을가해서 앞으로 나가게한다.
->(총알:게임오브젝트)에 (강체:리지드바디)에 힘을 가해서 (앞:z축)으로 나가게한다

총알을 맞으면 몬스터가 삭제된다. -> 게임이 재미가없다 -> 체력을 추가한다.  -> 공격력이 필요하다. -> 공격력? -> 총알에 추가하는 것이 구현하기 쉽다.

오브젝트: 
에너미무브먼트 이동금지

*각 상태를 불변수로 만들어 상태를 관리하기 -> 중첩상태가 존재할때 전반적은 코드의 작동을 이해하기 어려워 코들들 작성하는데 어려움이 있다.
오토: 조작하지않아도 이동하는 것.
정찰: 웨이포인트를 지정하면 이동하는 것.
가고일(정찰X): 추적대상이 사라지면 작동이 안됨.
이동: 타겟이 없어야 끝난다.

*유한상태: 어떤 상태가 언젠가는 끝난다. -> 상태가 중첩되어있으면 생각하기 어려워므로 중접된상태를 일일히 생각하기보단 정확하게 끝나는 상태를 기반으로 코드를 작성함.
몬스터가 플레이어를 발견했다. -> 시야안에 들어온다.
몬스터가 플레이어를 추적한다. -> 플레이어의 방향으로 이동한다.
몬스터가 플레이어를 공격한다. -> 공격이 쿨타임(딜레이) 끝나면 공격한다?
몬스터가 공격할 플레이어가 사라지면 -> 원래위치로 돌아간다.
정찰: 몬스터가 지정된 A위치에 도착하면 B로 이동하고, B위치에 도착하면 A로 이동한다. -> 웨이포인트로 이동하고 다음 지정된 웨이포인트로 이동한다. -> 랜덤으로 이동할 웨이포인트가 결정된다.

[GUI상호작용]

도착위치에 히트박스에 플레이어가 히트되면, 페이드인하여 엔딩화면을 보여준다.
